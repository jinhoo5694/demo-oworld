# PRD: 대전 오월드 랜딩 페이지 리뉴얼

## 1. 프로젝트 개요

### 1.1 목적
대전 오월드 공식 웹사이트의 랜딩 페이지를 현대적인 디자인으로 리뉴얼하여 사용자 경험을 개선하고, 모바일/태블릿/데스크톱 모든 디바이스에서 최적화된 반응형 웹을 제공합니다.

### 1.2 범위
- **포함**: 메인 랜딩 페이지의 UI/UX 디자인 전면 개편
- **제외**: 백엔드 로직 변경, 서브 페이지 리뉴얼, 예매 시스템 개편
- **유지**: 기존 모든 기능 및 콘텐츠 구조

### 1.3 목표
- 모던하고 직관적인 디자인 시스템 구축
- 모든 디바이스에서 일관된 사용자 경험 제공 (반응형)
- 웹 접근성 표준(WCAG 2.1 AA) 준수
- 페이지 로딩 속도 개선 (3초 이내)
- 모바일 우선(Mobile-First) 디자인 적용

## 2. 현재 상태 분석

### 2.1 기존 사이트 구조
**주요 네비게이션**
- 오월드?
- 이용정보
- 축제 및 이벤트
- 즐길거리
- 예매하기

**콘텐츠 섹션**
1. 메인 비주얼 슬라이더 (히어로 섹션)
2. OWORLD&EVENT - 이벤트 배너 6개 회전
3. NEWS&NOTICE - 공지사항 3개 항목
4. POPUPZONE - 프로모션 배너 4개
5. SNS 링크 (Facebook, Blog, Instagram, YouTube, KakaoTalk)
6. 운영 정보 (운영시간, 나이트 유니버스, 사파리 시간, 고객센터)

**주요 기능**
- 본인인증 시스템 (아이핀, SMS, 계좌, 신용카드, 공인인증서)
- OnePass 통합 로그인
- 검색 기능
- 자동 슬라이더 (10초 간격)

### 2.2 개선이 필요한 부분
- 레거시 브라우저(IE 8-11) 의존성
- 모바일 반응형 디자인 부족
- 과다한 팝업 및 배너로 인한 시각적 혼란
- 웹 접근성 표준 미준수
- 느린 페이지 로딩 속도
- 구시대적인 UI 디자인

## 3. 디자인 요구사항

### 3.1 디자인 원칙
1. **현대적이고 친근한 디자인**: 동물원/워터파크의 즐거운 이미지 전달
2. **콘텐츠 우선**: 불필요한 장식 제거, 핵심 정보 강조
3. **직관적인 네비게이션**: 3클릭 이내 목적지 도달
4. **시각적 계층 구조**: 중요도에 따른 명확한 정보 배치
5. **브랜드 일관성**: 오월드 브랜드 컬러 및 아이덴티티 유지

### 3.2 컬러 팔레트
- **Primary**: 오월드 브랜드 컬러 (파란색/초록색 계열 추정)
- **Secondary**: 포인트 컬러 (주황색/노란색 - 활기찬 느낌)
- **Neutral**: 회색 계열 (텍스트, 배경)
- **White/Dark**: 화이트 배경, 다크 텍스트

### 3.3 타이포그래피
- **제목**: 굵고 임팩트 있는 폰트 (Pretendard Bold, Noto Sans KR Bold)
- **본문**: 가독성 높은 폰트 (Pretendard Regular, Noto Sans KR Regular)
- **크기**: 반응형 (모바일 14-16px / 데스크톱 16-18px)

### 3.4 레이아웃 시스템
- **그리드**: 12컬럼 그리드 시스템
- **브레이크포인트**:
  - Mobile: 320px ~ 767px
  - Tablet: 768px ~ 1023px
  - Desktop: 1024px ~ 1920px+
- **컨테이너 최대폭**: 1440px

## 6. 기술 요구사항

### 6.1 반응형 디자인
- **Mobile-First 접근**: 모바일부터 디자인 후 데스크톱 확장
- **Breakpoints**:
  - 320px (Small Mobile)
  - 768px (Tablet)
  - 1024px (Desktop)
  - 1440px (Large Desktop)
- **Flexible Images**: srcset, picture 태그 활용
- **Touch-Friendly**: 터치 타겟 최소 44x44px

### 6.2 브라우저 지원
- Chrome (최신 2버전)
- Firefox (최신 2버전)
- Safari (최신 2버전)
- Edge (최신 2버전)
- ~~Internet Explorer 지원 중단~~

### 6.3 성능 최적화
- **페이지 로딩 속도**: 3초 이내
- **이미지 최적화**: WebP 포맷, Lazy Loading
- **CSS/JS 최소화**: Minify, Bundle
- **캐싱 전략**: Browser Caching, CDN 활용
- **Core Web Vitals**:
  - LCP (Largest Contentful Paint) < 2.5s
  - FID (First Input Delay) < 100ms
  - CLS (Cumulative Layout Shift) < 0.1

### 6.4 접근성 (Accessibility)
- **WCAG 2.1 AA 준수**
- **키보드 네비게이션**: 모든 인터랙티브 요소 접근 가능
- **스크린 리더**: ARIA 태그 적용
- **색상 대비**: 최소 4.5:1 (텍스트), 3:1 (UI 요소)
- **Alt Text**: 모든 이미지에 대체 텍스트

### 6.5 SEO
- **시맨틱 HTML5**: header, nav, main, section, footer
- **메타 태그**: title, description, keywords
- **Open Graph**: SNS 공유 최적화
- **구조화된 데이터**: Schema.org (선택사항)

## 8. 애니메이션 및 인터랙션

### 8.1 페이지 전환
- **Fade-in**: 페이지 로드 시 요소 서서히 나타남
- **Scroll Animation**: 스크롤에 따라 요소 등장

### 8.2 호버 효과
- **카드 호버**: Scale(1.05), Shadow 증가
- **버튼 호버**: 배경색 변경, 밝기 조정
- **링크 호버**: Underline, 색상 변경

### 8.3 로딩
- **페이지 로딩**: 스피너 또는 스켈레톤 UI
- **이미지 로딩**: Lazy Loading with Placeholder

### 8.4 마이크로 인터랙션
- **좋아요/공유 버튼**: Bounce 효과
- **알림**: Toast 메시지
- **툴팁**: 정보 아이콘 호버 시 표시

## 9. 콘텐츠 가이드라인

### 9.1 이미지
- **히어로 이미지**: 1920x1080px (16:9), WebP/JPG, < 500KB
- **이벤트 썸네일**: 800x600px (4:3), WebP/JPG, < 200KB
- **아이콘**: SVG 포맷, 벡터 이미지

### 9.2 텍스트
- **제목**: 간결하고 임팩트 있게 (10자 이내)
- **설명**: 명확하고 구체적으로 (50자 이내)
- **CTA**: 행동 유도 문구 (예: "지금 예매하기", "자세히 보기")

### 9.3 비디오 (선택사항)
- **히어로 비디오**: MP4, 1920x1080px, < 5MB
- **자동 재생**: 음소거 상태로 자동 재생
